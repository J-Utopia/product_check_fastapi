# 구현 계획

## Goal

- 현재 모두투어 B2C API 기준으로 일정표 검수용 FastAPI 백엔드를 재구축한다.
- 단체번호 입력 시 실제 API를 호출해 정규화 JSON을 만들고, 기본 검수 규칙을 적용한다.
- GPTs 연동용 스키마와 지침, 최신 검증룰 문서를 함께 정리한다.

## Evidence

- `modetour_itinerary_inspector.md`의 기존 구조는 `API 데이터 수집 -> 정규화 -> 교차검증 -> 결과 출력`이다.
- `modetour_b2c_api_current_inventory.md`에 현재 기준 엔드포인트가 정리되어 있다.
- 2026-06-18 기준 실호출 검증 결과, 익명 호출은 `401 Unauthorized`이며 `modewebapireqheader` 기반 헤더 캡처 후 `200 OK`를 확인했다.

## Scope

- `GetPackageInfo`
- `GetScheduleList`
- `GetProductDetailInfo`
- `GetHotelList`
- `GetFlightRemarkList`
- `GetProductKeyPointInfo`
- `GetPackageCouponList`
- `/run-itinerary` FastAPI 엔드포인트
- 최신 검증룰 JSON 5종
- GPTs 스키마/지침 md 문서

## Constraints

- 모든 파일은 `Python/ModewareBusiness/일정표검수` 폴더 안에서만 생성한다.
- 검증 없는 완료는 금지한다.
- 현재 API는 인증 헤더가 필요하므로 자동 헤더 캡처 또는 환경변수 주입이 가능해야 한다.
- 일반 상식 기반 판단 대신 API 교차 근거가 있는 규칙부터 우선 구현한다.

## Approach

1. Playwright로 모두투어 페이지 요청 헤더를 캡처한다.
2. 캡처 헤더로 현재 B2C API를 호출한다.
3. 각 엔드포인트 응답을 하나의 정규화 모델로 합친다.
4. 제목, 기간, 항공, 노팁/노쇼핑/노옵션, 숙박, 미팅 등 핵심 규칙을 결정적으로 검수한다.
5. 결과를 GPTs가 바로 소비할 수 있는 JSON 구조로 반환한다.
6. 동일 규칙을 사람이 읽기 좋은 JSON/Markdown 문서로 정리한다.

## Validation

- 정상 케이스: 정규화가 성공하고 정상 구조 보호 규칙이 동작하는지 테스트
- 실패 케이스: 제목 박수/일수 불일치, 노쇼핑/노옵션 모순 등 오류 검출 테스트
- 경계 케이스: `노팁 + 가이드/기사 경비 포함`을 오류로 만들지 않는지 테스트
- `pytest`
- `mypy`

## Repair Strategy

- 인증 캡처 실패 시 환경변수 기반 헤더 주입으로 우회 가능하도록 설계한다.
- API 응답 필드가 비어 있으면 즉시 실패 응답으로 내리고 추론 단계에 진입하지 않는다.
- 테스트 실패 시 실패 케이스 하나씩 분리해 원인 규칙과 정규화 계층을 수정한다.
