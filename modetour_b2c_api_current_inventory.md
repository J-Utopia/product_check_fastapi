# 모두투어 B2C 패키지 API 현황 및 응답 정보 정리

> 작성 목적: 현재 확인된 모두투어 B2C API 목록과 각 API에서 조회 가능한 정보를 정리한다.  
> 본 문서는 프로젝트 목표, 구현 구조, 코딩 방식, 검수 로직을 정의하는 문서가 아니다.  
> 이후 별도의 Codex 작업 요청 문서를 작성할 때 참고할 API 기초 자료로 사용한다.

---

## 1. 공통 기준

현재 확인한 API는 모두 단체번호를 기준으로 조회한다.

### 공통 파라미터

```text
productNo
```

예시:

```text
105195679
```

`105195679`는 특정 상품만을 분석하기 위한 고정값이 아니라 API 응답 구조를 확인하기 위해 사용한 예시 단체번호이다.

실제 상품마다 다음 내용은 달라질 수 있다.

- 여행 지역
- 여행 일수
- 일정 개수
- 항공 구간 수
- 호텔 수
- 방문 국가 및 도시 수
- 관광 일정 수
- 선택관광 수
- 식사 정보
- 쇼핑 정보
- 안내 문구 길이
- 상품 핵심포인트 내용
- 특이사항
- 쿠폰 유무

유럽, 미주, 중남미, 아프리카 등 장거리 상품은 일정이 길고 국가·도시·항공·호텔 이동이 많아 API 응답 내용이 매우 길어질 수 있다.

## 2.1 GetPackageInfo

### 요청 URL

```text
https://b2c-api.modetour.com/Package/GetPackageInfo?productNo=105195679
```

### 파라미터

```text
productNo = 단체번호
```

상품의 대표적인 기본정보를 확인할 수 있는 API이다.

- 출발 항공편명
- 단체번호
- P코드
- 전체 상품코드
- 상품명

- 도착일
- 도착시간
- 귀국 또는 리턴시간
- `arrivalWeekday`

## 2.2 GetScheduleList

### 요청 URL

```text
https://b2c-api.modetour.com/Package/GetScheduleList?productNo=105195679
```

### 파라미터

```text
productNo = 단체번호
```

### API 성격

상품의 상세 일정표를 확인할 수 있는 가장 중요한 API이다.

여행 일차별 일정, 항공 이동, 관광, 호텔, 식사, 옵션, 기타 활동 등이 포함되어 있다.

### 핵심 배열

```text
scheduleItemList
```

`scheduleItemList`에는 여행 일차별 정보가 배열 형태로 들어 있다.

예시 상품에서 배열 인덱스가 `0~4`까지 존재한다면 총 5개의 일차 정보가 있다는 의미이다.

다만 실제 일차 판단은 배열 인덱스보다 각 항목의 `first` 값을 기준으로 보는 것이 더 정확하다.

### 확인된 주요 필드

- `first`
- `date`
- `scheduleHotel`
- `listAirRouteInfo`
- `listGuidePlace`
- `listHotelPlace`
- `ortherActions`
- `placeHeader`
- `allPlaceTravelToday`
- `itiSeq`

### `first`

`first`는 해당 일정이 몇 일차인지 나타내는 값이다.

예시:

```text
first: 1
```

의미:

```text
1일차
```

```text
first: 5
```

의미:

```text
5일차
```

### `date`

해당 일차의 실제 날짜를 나타낸다.

예시:

```text
2026-06-24T00:00:00
```

### `scheduleHotel`

해당 일차의 예정 호텔 또는 호텔 관련 안내가 들어 있다.

예시:

```text
총 1개의 예정 호텔이 있습니다.
확정 되는대로 출발 1일 전까지 홈페이지를 통해 안내합니다.
```

`scheduleHotel`은 실제 확정 호텔명이 아니라 예정 호텔 안내 문구일 수 있다.

마지막 일차의 `scheduleHotel`이 `null`이고 전체 일정이 5일이라면 일반적인 4박 5일 일정일 가능성이 있다.

하지만 다음 예외가 있을 수 있으므로 `scheduleHotel`의 존재 여부만으로 숙박 수를 확정하면 안 된다.

- 기내박
- 선박박
- 열차박
- 야간 이동
- 호텔 미정
- 일정 변경
- 당일 일정
- 호텔 안내 문구만 존재하는 경우

### `itiSeq`

`itiSeq`는 같은 일차 안에서 일정이 진행되는 순서를 나타내는 값이다.

예시:

```text
itiSeq: 1
```

의미:

```text
해당 일차의 첫 번째 일정
```

```text
itiSeq: 2
```

의미:

```text
해당 일차의 두 번째 일정
```

```text
itiSeq: 4
itiSeq: 6
```

의미:

해당 일정들이 일차 안에서 각각 네 번째, 여섯 번째 순서에 위치한다는 의미로 볼 수 있다.

일정 내용은 다음 중 하나일 수 있다.

- 항공 이동
- 관광
- 호텔
- 식사
- 선택관광
- 쇼핑
- 자유시간
- 가이드 미팅
- 도시 이동
- 공항 이동
- 기타 안내

### `listAirRouteInfo`

항공 이동 또는 항공 구간 관련 정보가 포함될 가능성이 높은 배열이다.

추가 확인 대상:

- 항공사
- 편명
- 출발 공항
- 도착 공항
- 출발시간
- 도착시간
- 이동일
- 경유 여부
- 소요시간
- 구간 순서

### `listGuidePlace`

관광지, 방문 장소, 가이드 일정과 관련된 정보가 포함될 수 있다.

추가 확인 대상:

- 관광지명
- 도시명
- 국가명
- 관광 설명
- 가이드 설명
- 소요시간
- 입장 여부
- 포함 또는 불포함 여부
- 필수 관광 여부

### `listHotelPlace`

호텔 방문, 숙박, 체크인, 체크아웃 등 호텔 일정과 관련된 정보가 포함될 수 있다.

추가 확인 대상:

- 호텔명
- 숙박 도시
- 체크인
- 체크아웃
- 호텔 이동
- 예정 또는 확정 상태
- 호텔 등급

### `ortherActions`

기타 일정이 들어 있는 배열이다.

필드명이 `otherActions`가 아니라 실제 응답에서 `ortherActions`로 존재하는지 정확히 확인해야 한다.

포함 가능성이 있는 정보:

- 식사
- 이동
- 쇼핑
- 자유시간
- 선택관광
- 미팅
- 행사
- 기타 일정 안내

### `placeHeader`

일정 구간이나 장소를 구분하는 제목 또는 헤더 성격의 값일 수 있다.

정확한 의미는 여러 상품 응답과 실제 일정표 화면을 비교해 확인해야 한다.

### `allPlaceTravelToday`

해당 일차의 전체 방문 장소 또는 전체 이동 장소 목록으로 볼 수 있다.

정확한 배열 구조와 내부 필드는 실제 전체 JSON 확인이 필요하다.

### 긴 일정 상품에 대한 주의사항

예시 상품은 5일 일정이지만 유럽, 미주 등 장기 상품은 다음과 같이 응답이 훨씬 길어질 수 있다.

- `scheduleItemList`가 10개 이상
- 한 일차 안의 일정 항목이 수십 개
- 여러 국가와 도시 이동
- 항공편이 여러 구간
- 호텔이 일차마다 변경
- 기차, 버스, 선박 이동 포함
- 관광 설명이 장문 HTML로 제공
- 포함식, 불포함식, 자유식이 혼합
- 선택관광과 쇼핑 일정이 다수 존재

따라서 API 구조를 확인할 때 짧은 상품 하나만 기준으로 판단하면 안 된다.

다음 유형의 상품도 함께 확인해야 한다.

- 3일 이하 단기 상품
- 5일 내외 일반 상품
- 10일 이상 장기 상품
- 다국가 유럽 상품
- 미주 횡단 상품
- 항공 경유 상품
- 기내박 포함 상품
- 선박 또는 열차 숙박 상품
- 자유일정 포함 상품
- 선택관광이 많은 상품

## 2.3 GetProductDetailInfo

### 요청 URL

```text
https://b2c-api.modetour.com/Package/GetProductDetailInfo?productNo=105195679
```

### 파라미터

```text
productNo = 단체번호
```

### API 성격

상품의 상세 설명과 핵심 정보를 폭넓게 제공하는 API이다.

일정표 상단의 상품 핵심포인트, 상품 특징, 특이사항, 마일리지, 항공, 지역, 가이드 등의 정보를 확인할 수 있다.

### 현재 확인된 주요 필드

#### `accumulationExpectedTourMileage`

예시:

```text
6990
```

의미:

```text
예상 투어마일리지 적립금액
```

#### `actualPlaceSafetyGuidelineDisplayOrNot`

예시:

```text
Y
```

필드명 기준으로는 현지 또는 실제 장소 안전 가이드라인 표시 여부에 가까워 보인다.

이 값을 가이드 동행 여부로 바로 해석하면 안 된다.

가이드 동행 여부는 별도의 가이드 관련 필드와 비교해야 한다.

#### `addFareSingleRoom`

예시:

```text
300000
```

의미:

```text
독실 또는 싱글룸 사용 시 추가요금
```

#### `arrivalCity`

예시:

```text
DSN
```

의미:

```text
현지 도착 도시 코드
```

#### `arrivalCityName`

예시:

```text
오르도스
```

의미:

```text
현지 도착 도시명
```

#### `category1`

예시:

```text
아시아
```

#### `category2`

예시:

```text
중국
```

#### `category3`

예시:

```text
null
```

지역 분류 체계로 보이며 대분류, 중분류, 세부분류에 해당할 가능성이 있다.

#### `confirmWantedContent`

예시:

```text
1. 가이드명 : 조인철 (남)
```

단체별 확정 요청사항, 가이드 정보, 상품 관련 추가 안내 등이 포함될 수 있다.

실제 필드명이 정확히 `confirmWantedContent`인지 확인이 필요하다.

#### `groupBriefKeyword`

예시:

```text
#내몽고 #인컨타라사막 #오르도스초원 #사막숙박 #게르숙박 #별빛힐링
```

상품의 주요 특징과 여행 콘셉트를 나타내는 키워드이다.

#### `keyword`

별도의 검색용 또는 상품 분류용 키워드가 존재할 수 있다.

#### `productName`

상품명이다.

#### `productTypeCodeKey`

예시:

```text
M3001
```

상품 유형 또는 상품 분류를 나타내는 코드이다.

### 확인 가능한 상세 영역

- 단체별 특이사항
- 상품 POINT
- 스페셜 혜택
- 관광

### 추가 확인이 필요한 정보

```text
https://b2c-api.modetour.com/Package/GetHotelList?productNo=105195679
```

### 파라미터

```text
productNo = 단체번호
```

### API 성격

상품에서 이용할 예정이거나 확정된 호텔 정보를 확인할 수 있는 API이다.

### 현재 확인된 정보

- 예정 호텔 정보

### 전체 응답에서 추가 확인해야 할 값

- 호텔명
- 영문 호텔명
- 호텔 등급

# 5. 현재 시점에서 반드시 유의할 사항

1. `105195679`는 예시 단체번호이다.
2. 특정 상품 하나의 구조만으로 전체 상품 구조를 확정하면 안 된다.
3. 짧은 일정과 긴 일정의 응답 구조가 다를 수 있다.
4. 상품 유형에 따라 일부 배열이나 필드가 없을 수 있다.
5. `null`, 빈 문자열, 빈 배열을 각각 구분해서 확인해야 한다.
6. 필드명이 자연스러운 영어와 다를 수 있다.
7. `ortherActions`와 같은 오탈자 형태의 필드명이 실제로 존재할 수 있다.
8. 같은 의미의 정보가 여러 API에 중복 존재할 수 있다.
9. 같은 필드라도 상품에 따라 값의 형태가 다를 수 있다.
10. HTML 문자열이 포함될 수 있다.
11. 긴 일정은 응답 크기가 매우 클 수 있다.
12. 사용자가 확인한 필드 외의 값도 전체 응답에서 추가 조사해야 한다.
13. 필드 의미는 이름만 보고 확정하지 말고 실제 화면과 비교해야 한다.
14. 일정표 검수에 활용 가능한 값인지 별도로 판단해야 한다.
15. 새로운 상품 유형을 확인할 때마다 필드 목록을 확장해야 한다.
